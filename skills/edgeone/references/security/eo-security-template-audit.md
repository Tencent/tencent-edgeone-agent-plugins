# eo-security-template-audit

用于盘查 **EdgeOne 安全策略模板覆盖范围**，找出未绑定模板的域名，识别潜在的安全防护空白。该技能的核心目标是：

- 枚举当前站点下所有安全策略模板
- 识别哪些域名已绑定模板、哪些域名尚未绑定
- 对未覆盖的域名给出明确提示，辅助人工决策

## 适用输入

- 哪些域名没有绑安全模板
- 帮我检查模板覆盖情况
- 看看有没有域名漏绑了安全策略
- 帮我列出所有未绑定安全模板的域名

## 前置条件

### 认证与 CLI 前置动作

1. 所有腾讯云 API 调用统一通过 `tccli` 执行。
2. 如果环境中尚未配置可用凭证，必须先引导用户执行 [auth.md](./reference/auth.md) 中说明的登录方式：

```sh
tccli auth login
```

3. 需要明确告知用户：执行 `tccli auth login` 后，终端会打印授权链接，并在用户完成浏览器授权前保持阻塞；授权成功后命令会自动结束。
4. 严禁向用户索要 `SecretId` / `SecretKey`，也不要执行可能暴露凭证内容的命令（尤其是 tccli configure list）。

### 业务前置条件

1. 在确认站点参数并执行后续查询前，必须先确认 `tccli` 已处于可用登录状态。
2. 在确认 `tccli` 可用后，再确认目标站点的 `ZoneId`。
3. 如果用户只提供了域名，而当前上下文无法唯一定位到站点，应在确认 `tccli` 可用后，再要求用户补充 `ZoneId` 或其他可唯一定位站点的信息。
4. 本技能仅执行只读查询，不会对任何配置进行修改。

## 执行流程

### 1. 检查认证状态

优先确认当前环境中的 `tccli` 是否已登录且可用。

如果当前环境尚未登录或调用失败提示凭证不可用，先引导用户完成 [auth.md](./reference/auth.md) 中的浏览器授权登录，再继续后续步骤。

### 2. 确认范围

在确认 `tccli` 可用后，再确认本次盘查的最小必要信息：

- 站点标识
- `ZoneId`
- 是否需要盘查全部域名，还是仅针对特定子域名范围

如果缺少 `ZoneId`，先补齐站点信息，再进入采集阶段。

### 3. 查询接口文档（按需）

如果对 `teo` 相关接口的参数、字段、数据结构或返回含义不确定，应按照 [ref.md](./reference/ref.md) 中的方法先检索文档，再执行 API 调用，不要臆测字段含义。

典型检索方式包括：

- 检索 `teo` 服务的接口列表
- 阅读对应 Action 文档确认参数
- 阅读数据结构文档确认字段含义

### 4. 顺序采集数据

围绕同一个 `ZoneId`，按以下顺序采集数据：

**第一步：获取所有安全策略模板列表**

```sh
tccli teo DescribeWebSecurityTemplates --ZoneId <ZoneId>
```

**第二步：逐一获取每个模板的详细信息**

对第一步返回的每个模板 ID，依次执行：

```sh
tccli teo DescribeWebSecurityTemplate --ZoneId <ZoneId> --TemplateId <TemplateId>
```

**第三步：查询模板绑定关系**

```sh
tccli teo DescribeSecurityTemplateBindings --ZoneId <ZoneId> --TemplateId <TemplateId>
```

如果接口返回提示参数不完整、权限不足、字段名不匹配或其他异常，应按以下顺序处理：

- 先说明失败原因
- 如果是参数不完整，先执行 `tccli teo <Action> help` 获取该接口所需参数；如果返回信息仍不够清晰，再执行 `tccli teo <Action> help --detail` 获取详细说明
- 如果仍有疑问，再根据 [ref.md](./reference/ref.md) 的方法查询对应接口文档
- 补充必要参数后重新调用

### 5. 交叉比对，识别未覆盖域名

在完成采集后，将模板绑定关系与站点下的域名列表进行交叉比对：

- 汇总所有已绑定模板的域名
- 识别未出现在任何模板绑定关系中的域名
- 对未绑定域名按风险程度排序（如对外暴露的主域名优先提示）

### 6. 生成结论

先输出结论，再给出支撑证据，结构参见"输出要求"。

## 数据采集与分析方法

### 1. 模板列表总览

基于 `DescribeWebSecurityTemplates`：

- 汇总模板名称、模板 ID、模板类型、绑定域名数量、最近修改时间（如果返回结果中有）
- 关注是否存在空模板（已创建但未绑定任何域名）
- 关注是否存在命名不规范或用途不明的模板

### 2. 模板详情核查

基于 `DescribeWebSecurityTemplate`：

- 确认每个模板的防护规则是否完整、是否处于启用状态
- 标记防护规则为空或全部关闭的模板，即使已绑定域名也属于防护空白
- 关注模板中是否存在过于宽松的放行规则

### 3. 绑定关系分析

基于 `DescribeSecurityTemplateBindings`：

- 枚举每个模板绑定的域名列表
- 汇总所有已绑定域名的集合
- 与站点下的完整域名列表对比，得出未绑定域名清单

## 未覆盖域名判断规则

满足以下任一条件的域名，应标记为**未受安全模板保护**：

- 未出现在任何模板的绑定关系中
- 绑定的模板防护规则为空或全部处于关闭状态
- 绑定的模板已被删除或处于异常状态

对于未覆盖域名，按以下维度进行风险分级：

| 风险等级 | 判断依据 |
|---|---|
| 🔴 高风险 | 对外暴露的主域名或业务核心域名，完全未绑定任何模板 |
| 🟡 中风险 | 子域名或测试域名未绑定模板，但仍对外可访问 |
| ⚪ 待确认 | 无法判断域名用途，需人工确认是否需要防护 |

## 输出要求

最终必须用中文输出，优先给结论，再给证据，结构固定如下：

### 1. 盘查范围

- 站点标识
- `ZoneId`
- 盘查时间
- 数据来源接口

### 2. 模板覆盖摘要

- 当前模板总数
- 已绑定域名的模板数 / 空模板数
- 已覆盖域名总数
- **未覆盖域名总数**（重点标出）

### 3. 未覆盖域名清单

按风险等级分组列出未绑定模板的域名，格式示例：

```
🔴 高风险（需立即处理）
  - example.com

🟡 中风险（本周内处理）
  - test.example.com
  - staging.example.com

⚪ 待确认（需人工判断）
  - internal.example.com
```

### 4. 模板状态摘要

列出所有模板的基本信息及绑定状态：

| 模板名称 | 模板 ID | 绑定域名数 | 防护状态 | 备注 |
|---|---|---|---|---|
| ... | ... | ... | ... | ... |

### 5. 处理建议

- 立即处理项：高风险未覆盖域名，建议绑定合适的安全模板
- 本周内处理项：中风险未覆盖域名，建议评估后绑定
- 持续观察项：空模板或防护规则异常的模板，建议人工复核

## 工作原则

- 不要凭空判断域名的风险等级，必须结合域名命名、暴露情况等可观测信息给出依据
- 如果无法获取站点完整域名列表，应明确说明"当前仅能基于模板绑定关系输出已知覆盖情况，无法确认是否存在遗漏域名"
- 如果接口字段很多，优先提炼对安全运营有意义的信息，不要把整份原始 JSON 直接堆给用户
- 如果 API 调用失败，需要明确说明是 `ZoneId` 缺失、权限不足、参数错误，还是接口返回异常
- 本技能只做只读查询，不执行任何绑定或修改操作；如果用户希望补绑模板，应明确告知需要人工在控制台操作或调用写接口，并提示相应风险
- 该技能应作为独立能力使用，所有外部引用仅限 `reference` 目录下的说明文件
