# eo-traffic-daily-report

每日定时查询 7 层/4 层流量趋势，自动生成带宽峰值、请求量、Top 域名、Top 地区的 Markdown 日报，支持多日趋势对比和单域名多维度下钻分析（国家、省份、IP、UA、状态码、URL Path、Referer 等）。

## 涉及 API

| Action | 说明 |
|---|---|
| DescribeTimingL7AnalysisData | 查询 7 层时序流量/带宽/请求量数据 |
| DescribeTopL7AnalysisData | 查询 7 层 Top 域名、Top 地区等维度排名数据 |
| DescribeTimingL4Data | 查询 4 层时序流量/带宽/连接数数据 |

> **命令用法**：本文档只列出 API 名称和流程指引。
> 执行前请通过 [api-discovery.md](../api/api-discovery.md) 中的方式查阅接口文档，确认完整参数和响应说明。
> 筛选条件的完整说明参考 [指标分析筛选条件说明](https://edgeone.ai/document/56985)。

### DescribeTopL7AnalysisData 可用维度

`MetricName` 决定统计维度，以下为完整的维度列表：

| 维度 | 流量指标（MetricName） | 请求数指标（MetricName） | 返回值含义 |
|---|---|---|---|
| 域名 | `l7Flow_outFlux_domain` | `l7Flow_request_domain` | 域名（如 `example.com`） |
| 国家/地区 | `l7Flow_outFlux_country` | `l7Flow_request_country` | ISO 3166-1 alpha-2 国家代码（如 `CN`、`US`） |
| 省份（中国大陆） | `l7Flow_outFlux_province` | `l7Flow_request_province` | 腾讯云省份代码（见下方映射表） |
| 状态码 | `l7Flow_outFlux_statusCode` | `l7Flow_request_statusCode` | HTTP 状态码（如 `200`、`404`、`503`） |
| URL Path | `l7Flow_outFlux_url` | `l7Flow_request_url` | 请求路径（如 `/api/v1/data`） |
| Referer | `l7Flow_outFlux_referers` | `l7Flow_request_referers` | Referer 来源页面 |
| 客户端 IP | `l7Flow_outFlux_sip` | `l7Flow_request_sip` | 客户端源 IP 地址 |
| User-Agent | `l7Flow_outFlux_ua` | `l7Flow_request_ua` | 完整 UA 字符串 |
| 设备类型 | `l7Flow_outFlux_ua_device` | `l7Flow_request_ua_device` | 设备类型（如 Mobile、Desktop） |
| 浏览器类型 | `l7Flow_outFlux_ua_browser` | `l7Flow_request_ua_browser` | 浏览器类型（如 Chrome、Safari） |
| 操作系统 | `l7Flow_outFlux_ua_os` | `l7Flow_request_ua_os` | 操作系统（如 Windows、iOS） |
| 资源类型 | `l7Flow_outFlux_resourceType` | `l7Flow_request_resourceType` | 资源类型（如 html、jpg、js） |

> **使用建议**：
> - 域名下钻分析时，按需选取维度组合，不必一次查询全部维度
> - 常用组合：国家 + 省份 + IP（定位访问来源）、状态码 + URL Path（定位异常请求）、Referer（分析流量入口）、UA（识别客户端类型）

## 数据规约

在处理 EdgeOne 数据分析 API 的返回值和生成报告时，必须遵守以下规约：

### 单位换算：1000 进制（SI 标准）

EdgeOne 统一使用 **1000 进制**（SI 标准），与网络/存储行业惯例一致：

| 原始单位 | 换算 | 目标单位 |
|---|---|---|
| Byte | ÷ 1,000 | KB |
| KB | ÷ 1,000 | MB |
| MB | ÷ 1,000 | GB |
| bps | ÷ 1,000 | Kbps |
| Kbps | ÷ 1,000 | Mbps |
| Mbps | ÷ 1,000 | Gbps |

> **严禁使用 1024 进制**。输出报告中所有流量和带宽数值必须按 1000 进制换算。

### 时间戳前归规则

API 返回的时序数据点采用**时间戳前归**：数据点打在时间戳 `T` 上，代表 **[T, T + interval)** 这个时段的聚合数据。

例如使用 `day` 粒度查询时，时间戳 `2026-03-23T00:00:00+08:00` 对应的数据代表 **03-23 全天（00:00 ～ 次日 00:00）** 的数据。

### 全量数据 vs 防护后数据

七层访问数据查询接口（`DescribeTimingL7AnalysisData`、`DescribeTopL7AnalysisData`）**默认返回防护后的干净流量/请求数**（即排除了被 Web 安全模块拦截/挑战的请求）。

- 当用户需要**数据报告**（日报、周报、趋势分析等）时，应给出**完整的全量数据**，需要在 `Filters` 中同时传入被安全处置和未被安全处置的数据：

```
--Filters '[{"Key":"mitigatedByWebSecurity","Operator":"equals","Value":["yes","no"]}]'
```

- 当用户明确只关心干净流量或安全拦截流量时，才按需筛选。

> `mitigatedByWebSecurity` 的可选值：`yes`（已被 Web 安全模块拦截/挑战）、`no`（未被处置的正常请求）。
> 同一筛选条件的多个值之间为 **Or** 关系，不同筛选条件之间为 **And** 关系。

### 域名筛选

在 `DescribeTimingL7AnalysisData` 和 `DescribeTopL7AnalysisData` 中，可通过 `Filters` 的 `domain` Key 筛选特定域名的数据：

```
--Filters '[{"Key":"domain","Operator":"equals","Value":["example.com"]},{"Key":"mitigatedByWebSecurity","Operator":"equals","Value":["yes","no"]}]'
```

> 注意：`domain` 筛选和 `mitigatedByWebSecurity` 筛选需同时传入（不同 Key 之间为 And 关系）。

### 国家/地区代码

国家/地区维度返回 [ISO 3166-1 alpha-2](https://www.iso.org/iso-3166-country-codes.html) 标准代码，常见映射：

| 代码 | 国家/地区 | 代码 | 国家/地区 | 代码 | 国家/地区 |
|---|---|---|---|---|---|
| CN | 中国 | US | 美国 | JP | 日本 |
| KR | 韩国 | SG | 新加坡 | DE | 德国 |
| GB | 英国 | FR | 法国 | AU | 澳大利亚 |
| CA | 加拿大 | BR | 巴西 | IN | 印度 |
| RU | 俄罗斯 | ID | 印度尼西亚 | TH | 泰国 |

> 完整代码表参考 ISO 3166-1 标准。报告中应同时展示代码和中文名称。

### 省份代码映射（中国大陆）

省份维度返回腾讯云内部省份代码（整数），需映射为省份名称。`-1` 表示海外。
完整映射参考 [腾讯云地域/运营商映射表](https://www.tencentcloud.com/document/product/228/6316#region.2Fisp-mappings)，常用映射如下：

| 代码 | 省份 | 代码 | 省份 | 代码 | 省份 | 代码 | 省份 |
|---|---|---|---|---|---|---|---|
| -1 | 海外 | 22 | 北京 | 1050 | 上海 | 4 | 广东 |
| 1442 | 浙江 | 120 | 江苏 | 121 | 安徽 | 122 | 山东 |
| 2 | 福建 | 1135 | 湖北 | 1465 | 江西 | 1466 | 湖南 |
| 182 | 河南 | 1177 | 河北 | 1051 | 重庆 | 1068 | 四川 |
| 153 | 云南 | 118 | 贵州 | 173 | 广西 | 1441 | 海南 |
| 86 | 内蒙古 | 1469 | 山西 | 1464 | 辽宁 | 1445 | 吉林 |
| 1155 | 西藏 | | | | | | |

> 报告中必须将代码映射为中文省份名称，不得直接输出数字代码。

### IP 段聚合建议

当按客户端 IP（`_sip`）维度查询时，Top IP 列表通常会出现同一 /24 段的多个 IP。报告中应额外提供 **IP 段聚合分析**：
- 将同一 /24 段（前 3 段相同）的 IP 聚合统计
- 展示每个 /24 段的合计流量和包含的 IP 数量
- 有助于识别 NAT 出口、CDN 回源 IP 池、代理集群等模式

### 流量单位自适应显示

报告中的流量值应根据大小自动选择合适的单位：

| 条件 | 显示格式 | 示例 |
|---|---|---|
| ≥ 1 GB (10⁹ Byte) | `xx.xx GB` | 10.39 GB |
| ≥ 1 MB (10⁶ Byte) | `xxx.xx MB` | 138.44 MB |
| ≥ 1 KB (10³ Byte) | `xxx.xx KB` | 716.78 KB |
| < 1 KB | `xxx B` | 494 B |

## 场景 A：生成指定日期的流量日报

**触发**：用户说"帮我生成昨天的流量日报"、"看看过去 24 小时的带宽峰值是多少"、"生成 2026-03-20 的流量报告"、"分析一下 example.com 的流量来自哪些国家/IP/UA"、"看看这个域名的状态码分布"。

### 流程

**第一步**：确认日报日期范围

- 若用户说"昨天"/"今天"，自动计算对应的起止时间（ISO 8601 格式）
- 默认时间范围为"昨日 00:00 至 23:59"
- `DescribeTimingL7AnalysisData` 有约 10 分钟数据延迟，查询时注意避开最近 10 分钟

**第二步**：采集 7 层时序数据

调用 `DescribeTimingL7AnalysisData`，查询以下指标：
- `l7Flow_outFlux`：EdgeOne 响应流量，单位 Byte（用于计算总流量）
- `l7Flow_outBandwidth`：EdgeOne 响应带宽，单位 bps（用于计算带宽峰值）
- `l7Flow_request`：访问请求数，单位次（用于计算总请求量）
- 建议使用 `hour` 粒度（`Interval=hour`），便于识别峰值时段
- **必须**传入 `Filters` 获取全量数据（参见「数据规约 > 全量数据 vs 防护后数据」）

**第三步**：采集 7 层 Top 数据

调用 `DescribeTopL7AnalysisData`，分别查询：
- `l7Flow_outFlux_domain`：按域名维度统计响应流量 Top（默认 Top 10）
- `l7Flow_outFlux_country`：按国家/地区维度统计流量 Top（默认 Top 5）
- 如站点域名较多，可适当增大 `Limit` 参数
- **必须**传入 `Filters` 获取全量数据（参见「数据规约 > 全量数据 vs 防护后数据」）

**第四步**：采集 4 层时序数据（如有 L4 代理）

调用 `DescribeTimingL4Data`，查询以下指标：
- `l4Flow_outBandwidth`：出向带宽峰值
- `l4Flow_connections`：并发连接数
- 如果用户站点未使用 4 层代理，可跳过此步骤并在日报中注明

**第五步**：汇总生成 Markdown 日报

将以上数据整理为结构化日报，包含：带宽峰值（及峰值时段）、总请求量、Top 域名表格、Top 地区表格、4 层数据摘要。

**输出建议**：以完整 Markdown 文档输出，包含标题、时间范围、核心指标摘要和详细表格。

### 域名下钻：多维度流量分布分析

**触发**：用户在查看日报/周报后，要求进一步分析某个具体域名的流量分布，例如"看看 example.com 的流量都来自哪些国家"、"分析一下这个域名的访问者 IP 和 UA"、"这个域名有哪些异常状态码"。

> 此子流程是场景 A / B 的按需延伸，当用户需要对特定域名进行深度分析时调用。

**第一步**：确认下钻参数

- **目标域名**：从用户指定或上一步 Top 域名排行中选取
- **时间范围**：沿用当前报告的时间范围，或用户另行指定
- **分析维度**：根据用户需求选取，常见组合如下：

| 分析目的 | 推荐维度 |
|---|---|
| 访问来源分析 | 国家/地区（`_country`）+ 省份（`_province`）+ 客户端 IP（`_sip`） |
| 客户端画像 | User-Agent（`_ua`）+ 设备类型（`_ua_device`）+ 浏览器（`_ua_browser`）+ 操作系统（`_ua_os`） |
| 请求特征分析 | 状态码（`_statusCode`）+ URL Path（`_url`）+ Referer（`_referers`） |
| 资源类型分析 | 资源类型（`_resourceType`） |
| 全面画像 | 以上全部按需组合 |

> 不必一次查询所有维度，按用户关注点选取即可。如用户说"全面分析"，建议选取：国家 + 省份 + IP + UA + 状态码 + URL Path + Referer。

**第二步**：按维度采集 Top 数据

对每个选取的维度，调用 `DescribeTopL7AnalysisData`：
- **必须**在 `Filters` 中同时传入 `domain` 筛选和 `mitigatedByWebSecurity` 全量筛选
- 流量指标和请求数指标可按需选择（通常优先查流量指标 `l7Flow_outFlux_*`，必要时补充请求数指标 `l7Flow_request_*`）
- `Limit` 建议设为 20，可根据用户需要调整
- 各维度查询可并行执行以提高效率

**第三步**：采集域名时序汇总数据（可选）

调用 `DescribeTimingL7AnalysisData`，同样在 `Filters` 中传入 `domain` 筛选，查询该域名的流量、带宽、请求数汇总值，作为报告的核心指标。

**第四步**：数据后处理

- **省份代码映射**：将腾讯云省份代码转换为中文省份名称（参见「数据规约 > 省份代码映射」）
- **国家代码标注**：ISO 3166-1 alpha-2 代码旁附上中文名称
- **IP 段聚合**：将同一 /24 段的 IP 合并统计（参见「数据规约 > IP 段聚合建议」）
- **状态码分类**：按 2xx/3xx/4xx/5xx 进行归类汇总。注意 EdgeOne 使用 520-599 范围的自定义状态码（如 566 为托管规则拦截、567 为自定义规则/速率限制/Bot 管理拦截），完整说明参考 [异常状态码参考](https://edgeone.ai/document/58009)
- **UA 归类**：识别主力客户端类型（如 Dart 应用、爬虫、浏览器等）

**第五步**：生成流量分布报告

按照下方「输出格式 > 域名下钻」模板输出报告，包含核心指标 + 各维度 Top 表格 + 综合洞察分析。

## 场景 B：多日趋势对比

**触发**：用户说"对比一下最近 7 天的流量趋势"、"这周和上周的带宽有什么变化"、"最近一周每天的请求量是多少"。

### 流程

**第一步**：确认对比时间范围

- 解析用户描述的时间范围，计算起止时间
- `DescribeTimingL7AnalysisData` 单次查询时间范围不超过 31 天

**第二步**：采集多日汇总数据

调用 `DescribeTimingL7AnalysisData`，使用 `day` 粒度（`Interval=day`），查询指标：
- `l7Flow_outFlux`：每日响应流量（用于计算总量和趋势）
- `l7Flow_outBandwidth`：每日带宽峰值
- `l7Flow_request`：每日请求总量
- **必须**传入 `Filters` 获取全量数据（参见「数据规约 > 全量数据 vs 防护后数据」）

**第三步**：计算趋势指标

对每日数据计算：
- 日环比变化率（与前一日对比的百分比变化）
- 周同比变化率（如时间范围超过 7 天）
- 峰值日和低谷日标记

**输出建议**：以趋势表格形式输出，包含日期、带宽峰值、日环比、请求总量、日环比，标注峰值日和异常波动。
